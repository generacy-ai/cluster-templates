# Quickstart: Testing the Duplicate Clone Fix

## Prerequisites

- Docker and Docker Compose installed
- VS Code with Dev Containers extension (for devcontainer mode testing)
- A test repo URL (e.g., `https://github.com/your-org/test-repo.git`)

## Testing Devcontainer Mode

1. Open the `standard/` folder in VS Code
2. When prompted, click "Reopen in Container"
3. Verify in the terminal:
   ```bash
   # Should show only ONE repo directory
   ls /workspaces/
   # Expected: just the repo name (e.g., "test-repo"), NOT "project"

   # Verify env var is set
   echo $WORKSPACE_DIR
   ```
4. Check logs for devcontainer detection:
   ```bash
   docker compose logs orchestrator | grep "Devcontainer mode"
   ```

## Testing Standalone Docker Compose Mode

1. Create `.env` from template:
   ```bash
   cp standard/.devcontainer/.env.template standard/.devcontainer/.env
   ```
2. Set required vars:
   ```bash
   echo "REPO_URL=https://github.com/your-org/test-repo.git" >> standard/.devcontainer/.env
   ```
3. Start the cluster:
   ```bash
   cd standard/.devcontainer
   docker compose up -d
   ```
4. Verify clone path:
   ```bash
   docker compose exec orchestrator ls /workspaces/
   # Expected: "test-repo" (derived from REPO_URL), NOT "project"
   ```

## Testing WORKSPACE_DIR Override

1. Set explicit override in `.env`:
   ```bash
   echo "WORKSPACE_DIR=/workspaces/custom-name" >> standard/.devcontainer/.env
   ```
2. Restart and verify:
   ```bash
   docker compose down && docker compose up -d
   docker compose exec orchestrator echo $WORKSPACE_DIR
   # Expected: /workspaces/custom-name
   ```

## Testing --config Flag

1. Ensure your test repo has `.generacy/config.yaml`
2. Check logs for config detection:
   ```bash
   docker compose logs orchestrator | grep "config"
   # Should show --config flag being used
   ```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Two repos in `/workspaces/` | Check `DEVCONTAINER` env var is set; verify resolve-workspace.sh is being sourced |
| "Multiple config" error | Ensure only one clone exists; check `WORKSPACE_DIR` is correctly resolved |
| Clone to wrong path | Verify `REPO_URL` is set correctly; check `basename` derivation |
